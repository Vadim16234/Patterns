Есть три класса: Square, Triangle Circle реализующие метод showGraphic() интерфейса Graphic.
Так же есть класс Artist, который будет делегировать методы других классов.
В классе Artist объявляем переменную graphics интерфейса Graphic. Создаем setter для этой переменной. Далее создаем
метод showGraphic() внутри которого, с помощью переменной graphics интерфейса Graphic вызываем метод showGraphic().
В клиентском классе DelegateRunner создаем объект класса Artist. Затем у переменной artist класса Artist
вызываем setter setGraphics() и передаем в качестве аргумента new Square() или другой аналогичный класс.
Теперь в переменной artist лежит новый объект класса Square. Вызываю у данной переменной доступный метод showGraphic().
Результат: так как переменная artist хранит в себе объект класса Square, вызванный метод showGraphic() будет вызван
из класса Square. В качестве аргумента можно передать другой аналогичный класс и методы будут вызываться
из других классов, тем самым происходит делегация.
